Quick Start

// 1. Build the WASM package
// wasm-pack build crates/wasm --target web --out-dir pkg

// 2. Import and initialize
import init, { CanvasEngine } from './pkg/rendero.js';
await init();

// 3. Create an engine and canvas
const engine = new CanvasEngine("My Canvas", 1);
const ctx = canvas.getContext('2d');
engine.set_viewport(1920, 1080);

// 4. Add shapes
engine.add_frame("Card", 100, 100, 400, 300, 1, 1, 1, 1);
engine.add_text("Title", "Hello Rendero", 120, 120, 32, 0, 0, 0, 1);
engine.add_rounded_rect("Button", 120, 200, 160, 44, 0, 0.44, 0.89, 1, 8);

// 5. Render
engine.render_canvas2d(ctx, 1920, 1080);

// That's it. Pan, zoom, select, undo/redo — all built in.

Spatial Grid Culling

O(1) viewport culling via spatial hash grid. Only visible nodes are processed, enabling millions of nodes.

Hierarchical LOD

Subtrees smaller than 50px on screen skip descendants entirely. Sub-pixel nodes are culled. Zero wasted draw calls.

Dual Render Paths

GPU-accelerated Canvas 2D for screen rendering. CPU tile-based rasterizer (64x64 L1-friendly tiles) for pixel-perfect export.

Document Model

Arena-based tree with frames, rectangles, ellipses, vectors, text, images, boolean ops, auto-layout, and grouping.

CRDT Collaboration

Operation-based CRDT with Lamport timestamps and fractional indexing. Ready for real-time multi-user editing.

494KB WASM

The entire engine compiles to ~494KB gzipped WASM. No runtime dependencies. Works in any modern browser.
